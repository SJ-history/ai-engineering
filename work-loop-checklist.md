# 작업 루프 체크리스트

> 작업을 시작할 때마다 이 순서를 따르세요. 에이전트가 길을 잃지 않게 하는 핵심입니다.

## 준비 (작업 시작 시 1회)
- [ ] 새 브랜치를 팠는가?  `git checkout -b feature/<작업이름>`  ← 안전벨트
- [ ] 이 작업의 "완료 기준"이 명확한가? (무엇이 되면 끝인지 한 문장으로)
- [ ] 검증 신호가 있는가? (아래 중 최소 1개)
  - [ ] 테스트 실행 가능 (`npm test` / `pytest` 등)
  - [ ] 타입체크/빌드 가능 (`tsc` / `build` 등)
  - [ ] 실제 실행해서 눈으로 확인 가능 (화면/응답/로그)

## 반복 루프 (작은 단위마다)
- [ ] 1. 작업을 작은 단위로 쪼갰는가?
- [ ] 2. 에이전트에게 **"한 단위만"** 시켰는가?
- [ ] 3. 변경 후 검증 신호로 확인했는가? (테스트/빌드/실행)
- [ ] 4-A. 통과 → 커밋하고 다음 단위로.  `git commit -m "<설명>"`
- [ ] 4-B. 실패 → 원인 파악 후 수정하고 다시 검증.
- [ ] 5. 같은 문제 3번 실패 → **멈추고** 상황 정리해서 판단(사람에게 질문).

## 마무리 (작업 종료 시)
- [ ] 전체 테스트/빌드가 통과하는가?
- [ ] 불필요한 디버그 코드/주석/로그를 지웠는가?
- [ ] 반복된 실수가 있었나? → `AGENTS.md`의 "금지사항"에 추가.
- [ ] 그 개선분을 개인 하네스 키트에도 반영했는가?

---
### 자주 쓰는 프롬프트 스니펫
- 레포 파악: "이 레포를 신입에게 설명하듯 구조/명령어/진입점을 근거와 함께 정리해줘."
- 작게 시키기: "지금은 <이 한 단위>만 구현하고, 끝나면 <검증방법>으로 검증한 결과를 보여줘."
- 막혔을 때: "지금까지 시도한 것과 각각 왜 실패했는지 정리하고, 대안 2가지를 제안해줘."
