# 04. 온보딩 플레이북 (Day 0 → Day 30)

> 새 조직·새 프로젝트에 투입됐을 때, 하네스와 루프를 **체계적으로** 세우는 실행 플레이북.
> `01`~`03`이 이론이라면 이 문서는 현장 실행 절차다.
> IDE·에이전트가 무엇이든(IntelliJ+Claude, VS Code+Copilot, Cursor 등) 절차는 동일하다.

전제: IDE와 에이전트는 설치·로그인 완료. 소스코드는 아직 미확보.

---

## Phase 0 — 착수 전 (Day 0)

**목표**: 개인 표준을 반입하고 에이전트 환경 기본값을 점검한다.

| 작업 | 세부 |
|------|------|
| 표준 반입 | 이 저장소를 clone → 재사용 자산 확보 |
| 도구 기본값 점검 | 자동 승인 정책(초심자는 off), git 연동, 터미널 연동 확인 |
| 인증 경계 확인 | 계정·자격증명이 올바른 조직에 연결됐는지 (오push 방지) |

산출물: 없음(준비 단계). `templates/AGENTS.md`를 손에 쥔 상태.

---

## Phase 1 — 정찰 (Day 1, 코드 확보 직후)

**목표**: 코드를 고치기 전에 **지도를 먼저 그린다.**

> 원칙: 첫날 리팩터링 금지. 이해가 통제보다 앞서야 한다.

에이전트에게 정찰을 위임한다 (예시 지시):
```
이 레포를 처음 온 시니어에게 인수인계한다고 가정하고 정리해줘:
1) 시스템 목적과 도메인   2) 아키텍처와 폴더별 책임
3) 빌드/실행/테스트/배포 명령   4) 진입점과 핵심 데이터 흐름
5) 기술 스택과 외부 의존성
반드시 README·설정파일·CI 정의를 근거로 인용해서 답해줘.
```

산출물: 정찰 노트 → Phase 2의 `AGENTS.md` 원재료.

---

## Phase 2 — 하네스 구축 (Day 1–3)

**목표**: 5대 기둥을 세운다. (상세 이론 → `02-harness-engineering.md`)

### 2-1. 컨텍스트 — 규칙 파일은 하나만
- `templates/AGENTS.md`를 레포 루트에 복사, 정찰 노트로 빈칸을 채운다.
- 에이전트가 `AGENTS.md`를 자동 인식하는지 확인한다. 못 읽는 에이전트라면
  "정본은 루트 AGENTS.md를 따른다"는 **한 줄짜리 포인터 파일**만 추가한다(본문 복제 금지).

### 2-2. 도구
- `AGENTS.md`에 적은 명령을 **터미널에서 실제로 하나씩 실행 검증**한다.
  (동작하지 않는 명령을 규칙에 적으면 에이전트가 헛발질한다.)
- 에러 메시지가 행동 가능한지 확인. 부실하면 개선 대상으로 기록.

### 2-3. 가드레일
- 브랜치 전략 확정: `git checkout -b feature/<작업>` 습관화
- 위험 작업 승인 게이트 설정 (삭제·force push·프로덕션·시크릿)

### 2-4. 관찰가능성 & 인터페이스
- 에이전트 행동 로그 확인 경로 파악
- 지시·보고 형식을 `AGENTS.md`의 "에이전트 작업 규약"으로 통일

**Phase 2 완료 기준**: 에이전트가 프로젝트 관습을 알고, 안전하게 되돌릴 수 있는 상태.

---

## Phase 3 — 루프 가동 (Day 3–7)

**목표**: 검증 신호를 연결하고 작은 루프를 돌린다. (상세 → `03-loop-engineering.md`)

- `templates/verification-matrix.md`로 작업 유형별 검증 신호를 매핑한다.
- 첫 실전 작업은 **의도적으로 작은 것**을 골라 루프를 검증한다:
  1. 완료 기준 정의 → 2. 검증 신호 확보 → 3. 작게 쪼개기
  → 4. 한 단위 실행·검증 → 5. 통과 시 커밋 / 실패 시 교정 → 6. N회 실패 시 에스컬레이션
- `templates/task-loop-checklist.md`를 곁에 두고 매 단위마다 확인한다.

**Phase 3 완료 기준**: "작게 고치고 → 기계로 검증"이 몸에 붙은 상태.

---

## Phase 4 — 굳히기 (Day 7–30)

**목표**: 하네스를 조직 지능으로 진화시키고, 개인 표준에 환류한다.

- 반복 관찰된 실수 → `AGENTS.md`의 "금지사항"에 추가 (하네스 강화)
- 반복 절차 → 스니펫·스크립트로 자동화
- 조직 특유의 좋은 패턴 → **이 개인 저장소에 역반영** → 다음 조직에서 더 빠른 착수
- `docs/05-maturity-model.md`로 현재 성숙도를 진단하고 다음 목표 레벨 설정
