# 06. 용어집 (Glossary)

이 표준에서 사용하는 핵심 용어의 정의. 조직마다 용어가 다를 수 있으므로 여기서 기준을 고정한다.

## 핵심 프레임워크

**하네스 엔지니어링 (Harness Engineering)**
에이전트가 작동하는 정적 환경(컨텍스트·도구·가드레일·관찰가능성·인터페이스)을 설계하는 활동.
"경기장을 만드는 일."

**루프 엔지니어링 (Loop Engineering)**
에이전트의 동적 피드백 사이클(계획→실행→관찰→검증→교정)을 설계하는 활동.
"경기장 위에서 방향을 교정하는 일."

**에이전트 루프 (Agentic Loop)**
LLM 에이전트가 목표 달성까지 반복하는 인지-행동 사이클. 각 바퀴마다 도구를 호출하고
결과를 관찰해 다음 행동을 결정한다.

## 하네스 관련

**컨텍스트 엔지니어링 (Context Engineering)**
에이전트에게 어떤 정보를 언제 제공할지 설계하는 활동. 프롬프트 엔지니어링의 상위 개념.

**계층적 컨텍스트 (Hierarchical Context)**
모든 정보를 한 번에 주지 않고, 항상 로드/조건부 로드/검색 기반으로 나눠 제공하는 원칙.

**가드레일 (Guardrail)**
에이전트의 행동을 안전 범위로 제한하는 경계(권한·격리·복구 가능성).

**복구 가능성 (Reversibility)**
에이전트의 변경을 안전하게 되돌릴 수 있는 성질. 브랜치·커밋·스냅샷으로 확보.

**결정성 (Determinism)**
같은 입력에 항상 같은 출력을 내는 성질. 도구가 결정적일수록 루프가 안정적이다.

**관찰 가능성 (Observability)**
에이전트가 무엇을 왜 했는지 추적·측정할 수 있는 성질.

## 루프 관련

**그라운드 트루스 (Ground Truth) / 검증 신호 (Verification Signal)**
작업의 성공 여부를 기계적으로 판정하는 현실 신호. 실행 결과 > 테스트 > 타입/컴파일 > 정적분석 순.

**짧은 피드백 루프 (Short Feedback Loop)**
변경과 검증 사이 간격이 짧은 상태. 실패 국소화와 저렴한 롤백을 가능케 한다.

**종료 조건 (Termination Condition)**
루프를 멈추는 규칙(재시도 상한, 예산 한계, 불확실성 게이트).

**에스컬레이션 (Escalation)**
에이전트가 스스로 해결 못 할 때 상황을 정리해 사람에게 넘기는 행위.

**자기 교정 (Self-Correction)**
실패 신호를 받아 롤백하거나 경로를 바꿔 재시도하는 능력.

**평가 (Evaluation / Eval)**
루프가 좋은 결과를 내는지 지표·골든 케이스로 반복 측정하는 활동.

**규칙 환류 (Rule Feedback)**
겪은 실패를 일회성 수정이 아니라 재사용 가능한 규칙(금지사항 등)으로 되먹이는 활동.

**검증자 분리 (Maker ≠ Checker)**
만드는 에이전트와 검사하는 에이전트를 분리해 자기 채점 편향(같은 주체가 자기 결과를
후하게 평가하는 경향)을 보정하는 기법.

**인지적 항복 (Cognitive Surrender)**
루프가 매끄러워질수록 사람이 결과물을 이해하려는 노력 없이 승인만 하게 되는 위험.
루프가 검증을 대체해준다는 착각에서 비롯되며, 하네스·루프 전체를 무력화하는 리스크.

**상태 파일 (State File / Progress File)**
세션이 끝나도 남는 진행 기록. "에이전트는 잊어도 저장소는 잊지 않는다"는 원칙을
구현하는 산출물. `templates/progress-template.md` 참조.

**오토메이션 (Automation, 루프 구성요소)**
정해진 주기·조건에 사람 개입 없이 루프를 시작하는 트리거(cron, hooks 등).

**워크트리 (Worktree)**
같은 저장소 안에서 작업마다 독립된 작업 공간을 제공해 동시 작업 간 충돌을 방지하는 기법.

**스킬 (Skill, 루프 구성요소)**
반복 설명이 필요 없도록 프로젝트 지식을 파일로 코드화한 것. `AGENTS.md`의 하위 계층에 해당.

## 산출물·표준

**AGENTS.md**
프로젝트 컨텍스트·규칙을 담는 **단 하나의 규칙 정본 파일**. 업계 공통 표준 파일명으로,
주요 코딩 에이전트가 레포 루트에서 자동 인식한다. 이 표준에서 규칙은 이 파일에만 존재한다.

**포인터 파일 (Pointer File)**
`AGENTS.md`를 직접 읽지 못하는 에이전트를 위해 두는 한 줄짜리 파일(예: `CLAUDE.md`).
"정본은 루트 AGENTS.md를 따른다"는 위임 선언만 담고, 규칙 본문은 복제하지 않는다.

**MCP (Model Context Protocol)**
에이전트와 외부 도구/데이터를 연결하는 개방형 프로토콜. 벤더 락인을 줄이는 표준 인터페이스.

**성숙도 모델 (Maturity Model)**
하네스·루프 수준을 L0–L4로 측정하는 척도. `05-maturity-model.md` 참조.
