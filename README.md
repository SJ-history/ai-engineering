# 하네스 키트 (Harness Kit)

새로운 회사/프로젝트에 도착했을 때, **에이전트(커서/클로드)가 길을 잃지 않고
내가 원하는 방향으로 일하게 만들기 위한 개인용 세팅 템플릿**입니다.

어느 회사, 어느 언어, 어느 프레임워크를 가든 이 폴더를 통째로 복사해서
그대로 재사용할 수 있게 만들어졌습니다.

---

## 이 키트는 무엇인가

- **하네스(Harness)** = 에이전트가 일하는 "판(환경)". 규칙 / 명령어 / 안전벨트 / 로그.
- **루프(Loop)** = 그 판 위에서 "작게 고치고 → 바로 검증"을 반복하는 사이클.

이 두 가지를 새 환경에서 빠르게 세팅하기 위한 파일들이 들어 있습니다.

## 파일 구성

| 파일 | 역할 | 어디에 두나 |
|------|------|------------|
| `AGENTS.md` | 에이전트에게 주는 프로젝트 규칙집 | 새 레포의 **루트**에 복사 |
| `cursor-rule.mdc` | 커서 전용 항상-적용 규칙 | 새 레포의 `.cursor/rules/` 폴더에 복사 |
| `work-loop-checklist.md` | 작업할 때마다 돌리는 루프 체크리스트 | 곁에 열어두고 매 작업마다 확인 |

---

## 이 키트는 독립된 git 저장소입니다
특정 프로젝트에 종속되지 않도록 별도 git 저장소로 관리합니다.
새 회사/PC에 가면 이 저장소를 clone(또는 복사)해서 그대로 재사용하세요.

```bash
# 원격에 올려두면 어디서든:
git clone <이 저장소 주소> ai-engineering
```

## 사용 순서 (새 회사 첫날 → 실전)

### STAGE 0. 코드 받기 전 (지금)
- 이 저장소를 안전한 곳(예: 개인 GitHub/GitLab)에 push 해두고, 새 환경에서 clone 한다.
- 커서 설정 점검: 자동승인 끄기(초보 추천), git 연동, 터미널 연동 확인.

### STAGE 1. 코드 받은 직후 — "고치기 전에 지도부터"
아직 코드를 수정하지 말고, 에이전트에게 레포 지도를 그리게 한다:

```
이 레포를 처음 보는 신입에게 설명한다고 생각하고 정리해줘:
1) 무슨 프로젝트인지  2) 폴더 구조와 역할  3) 빌드/실행/테스트 명령어
4) 진입점(main) 파일  5) 언어/프레임워크/DB
README나 설정파일(package.json, pyproject.toml 등)을 근거로 알려줘.
```

### STAGE 2. 하네스 세팅
1. `AGENTS.md`를 새 레포 루트에 복사하고, STAGE 1 결과로 빈칸을 채운다.
2. `cursor-rule.mdc`를 `.cursor/rules/`에 복사한다.
3. `AGENTS.md`에 적은 명령어들을 **터미널에서 실제로 한 번씩 실행**해 본다
   (안 도는 명령을 적어두면 에이전트가 헛발질함).
4. 작업 전 항상 새 브랜치: `git checkout -b feature/<작업이름>`  ← 안전벨트

### STAGE 3. 루프 세팅
- 검증 신호(테스트/타입·빌드/실행 결과) 중 최소 1개를 확보한다.
- 작업을 작은 단위로 쪼갠다.
- `work-loop-checklist.md`를 열어두고 매 단위마다 체크한다.

### STAGE 4. 굳히기
- 반복되는 실수를 발견하면 → `AGENTS.md`의 "금지사항"에 추가.
- 그 개선분을 **이 키트에도 역으로 반영** → 다음 회사에서 더 빠르게 세팅.

---

한 문장 요약: **고치기 전에 지도부터 그리고(1), 규칙과 안전벨트를 채우고(2),
작게 고쳐서 매번 기계로 검증하라(3).**
